# 事务

一组操作的集合，同时成功或同时失败

## 一、事务操作

### 1. 查看设置事务提交方式

```mysql
select @@autocommit
set @@autocommit = 0;
```

### 2. 提交事务

```mysql
commit;
```

### 3. 回滚事务

```mysql
rollback;
```

### 4.开启事务

```mysql
start transaction 或 begin
```

## 二、事务四大特性

ACID

* Atomicity：原子性
* Consistency：一致性：所有数据保持一致状态
* Isolation：隔离性：依赖隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
* Durability：持久性

## 三、并发事务问题

* 脏读：一个事物读到另一个事物还没有提交的数据
* 不可重复读：一个事物先后读取同一记录，但数据不同
* 幻读：一个事物按照条件查询数据时没有对应数据行，但插入时发现数据已经存在

```mysql
select @@TRANSACTION_ISOLATION
```

