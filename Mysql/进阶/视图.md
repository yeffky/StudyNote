# 视图

* 视图是一种虚拟存在的表，视图中的数据并不实际存在，行和列数据来自定义视图的查询中使用的表，并且实在使用过程时动态生成的

* 视图只保存查询的SQL逻辑，不保存查询结果

## 一、视图操作

### 1.创建视图

```mysql
create [or replace] view view_name as select语句 [with[cascaded|local] check option];
```

### 2.查询视图

```mysql
select * from view_name;
```

### 3.修改视图

```mysql
方式一： create [or replace] view view_name as select语句 [with[cascaded|local] check option];
方式二：alter view view_name as select语句 [with[cascaded|local] check option];
```

### 4.删除视图

```mysql
drop view view_name;
```

## 二、检查选项(递归检查)

当使用with check option创建时，MYSQL会通过视图检查更改的每个行，使其符合视图的定义。允许基于视图创建视图，会检查依赖视图中的规则保持一致性。有两个选项local和cascaded

### 1. cascaded

级联条件，要同时级联两个表进行约束，会在依赖视图增加一个check option，即会检查本视图以及依赖视图的条件。

### 2. local

不递归传递检查约束

## 三、视图的更新与作用

视图行要和基础表中的行一对一

### 1. 视图不可更新的情况

* 聚合函数或窗口函数
* distinct
* group by
* having
* union 或 union all

### 2. 视图的作用

* 操作简单，简化操作，不必为以后的操作每次指定全部条件
* 数据库可以授权，但不能到单行单列。通过视图用户只能查询和修改他们所能见到的数据
* 数据独立，帮助用户屏蔽基表结构的变化
